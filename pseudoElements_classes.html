Pseudo Elements Vs Pseudo Class Selectors

Pseudo-classes

Pseudo-classes select regular elements but under certain conditions, like when their position relative to siblings or when theyâ€™re under a particular state. They target some bit of the html document that is in a certain state for instance they are the first element of their type, or they are being hovered over by the mouse pointer. They tend to act as if you had applied a class to some part of your document.
Pseudo-classes are keywords that start with a colon: Here is a list of pseudo-classes in CSS3
Dynamic pseudo-classes
  • :link
  • :visited
  • :hover
  • :active
  • :focus
UI element states pseudo-classes
  • :enabled
  • :disabled
  • :checked
Structural pseudo-classes
  • :first-child
  • :nth-child(n)
  • :nth-last-child(n)
  • :nth-of-type(n)
  • :nth-last-of-type(n)
  • :last-child
  • :first-of-type
  • :last-of-type
  • :only-child
  • :only-of-type
Other pseudo-classes
  • :not(x)
  • :target
  • :lang(language)

Pseudo-elements

Pseudo-elements effectively create new elements that are not specified in the markup of the document and can be manipulated much like a regular element. This to create effects with minimal markup, also aids significantly in keeping the presentation of the document out of the HTML and in CSS where it belongs (separation of concern) .We use double colon (::) on pseudo-elements

  • ::after
  • ::before
  • ::first-letter
  • ::first-line
Unfortunately animations and transitions on pseudo elements are buggy or not implemented at the moment in most browsers.
