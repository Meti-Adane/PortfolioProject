Flex layout vs Grid layout

The basic difference between CSS Grid Layout and CSS Flexbox Layout is that flexbox was designed for layout in one dimension - either a row or a column. Grid was designed for two-dimensional layout - rows, and columns at the same time.

One dimensional vs two dimensional

flexbox is one-dimensional, while CSS Grid is two-dimensional. Flexbox lays out items along either on the horizontal or the vertical axis. It is most suitable for arranging elements in either a single row, or a single column.
On the other hand, CSS Grid lets you work along two axes: horizontally and vertically. Grid allows you to declare the sizing of rows and columns and then explicitly place things into both rows and columns as needed. Grid is most suitable for arranging elements in multiple rows and columns.

Flex wrap vs grid wrap

The beauty of Grid and Flexbox lies in the ability to stretch and squeeze the items based on the amount of space available. Flexbox achieves this using flex-grow and flex-shrink properties, and Grid achieves this using a combination of minmax and auto-fill functions inside the grid-template-columnsproperty.


While flexbox can make rows and columns in the sense that it allows elements to wrap, there‚Äôs no way to declaratively control where elements end up since the elements merely push along a single axis and then wrap or not wrap accordingly.
Additionally, in flexbox, everything is placed along a one-dimensional plane and it‚Äôs because of the single dimension property that we can do things like aligning elements along a baseline ‚Äî which is something grid is unable to do.

Overlapping items in Flexbox vs Grid

Getting elements to overlap in flexbox requires considering negative margins, transforms, or absolute positioning in order to break out of the flex behavior. With grid, we can place items on overlapping grid lines, or even right within the same exact grid cells.

Grid is Container-Based, Flexbox is Content-Based

In flexbox layout, the size of a cell (flex-item) is defined inside the flex-item itself, and in the grid layout, the size of a cell (grid-item) is defined inside the grid-container. flexbox layout is calculated after its content is loaded whereas the grid layout is calculated regardless of the content inside it. It is not recommended to use flexbox to build the overall layout of a website.

Flexbox has fairly good browser support, while CSS Grid is not supported by IE11- and Edge 15-.
